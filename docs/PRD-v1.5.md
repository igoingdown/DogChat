## 一、产品愿景

**一句话**：让遛狗时认识的狗友，不需要人类互相加微信，也能维持狗狗之间的社交关系。

**核心问题**：养狗人的社交困境是社交孤岛——狗狗有朋友圈，但狗主人之间缺乏低压力的连接方式。建微信群太重、太中心化、太尴尬。

**解决方案**：以狗为节点的去中心化社交广播网络。人是狗的表达代理人。狗友之间可以看到彼此狗狗的动态，但狗主人之间不需要互相认识。

---

## 二、目标用户

城市养犬人群。核心画像：每天遛狗 1-2 次，在小区/公园形成固定遛狗圈子，狗狗之间已建立线下玩伴关系。

冷启动：创始人的直接信任圈（约 10-30 位熟识的狗友），面对面推荐。

## 三、MVP 功能清单

### Must Have

| ID | 功能 | 说明 |
|---|---|---|
| M1 | 微信授权登录 | 一键登录，无需用户名密码 |
| M2 | 狗狗档案创建 | 名字 + 头像（相册选），系统生成 dogID |
| M3 | 狗友关系 | 双向确认：A 扫 B 的狗狗二维码 → B 确认 → 建立关系 |
| M4 | 狗狗圈 | 以狗的名义发动态（图文），仅狗友可见，按时间倒序 |
| M5 | 狗狗圈互动 | 点赞 + 评论（评论者显示为"XX的主人"） |
| M6 | 约遛狗 | 发起约遛邀请（时间+地点），狗友可见并响应 |
| M7 | 内容安全 | 接入微信内容安全 API（文本 + 图片），UGC 过审后发布 |

|---|---|---|

| S2 | 约遛日历 | 历史和即将到来的约遛记录 |
| S3 | 动态通知 | 新动态/评论的微信服务通知 |
| S4 | 狗狗资料补充 | 品种、年龄、性别、性格标签 |

### Could Have

| ID | 功能 | 说明 |
|---|---|---|
| C1 | 遛狗轨迹 | GPS 记录遛狗路线 |

## 四、核心交互

### 信息架构（底部 3 Tab）

1. **狗狗圈**（首页）— 狗友动态信息流
2. **约遛狗** — 发起/查看约遛
3. **我的** — 狗狗档案列表 → 单犬页面（资料 / 狗友列表 / 二维码）

### 关键流程

- **添加狗友**：我的 → 选狗 → 出示二维码 → 对方扫码 → 确认 → 双向绑定
- **发狗狗圈**：首页 → 发布 → 选以哪只狗身份发 → 写内容/选图 → 发布（内容安全审核）
- **约遛狗**：约遛 Tab → 发起 → 选时间地点 → 选邀请的狗友 → 发送

## 五、技术方案

### 5.1 技术栈

| 层 | 选型 | 说明 |
|---|---|---|
| 前端 | 微信小程序原生开发 | 无额外框架，最简路径 |
| 后端 | 微信云开发（Serverless） | Node.js 云函数 |
| 数据库 | 云数据库（JSON 文档型） | 内置，类 MongoDB |
| 文件存储 | 云存储 | 狗狗照片、动态图片 |
| 登录 | `wx.login()` | 免鉴权获取 openid |
| 内容安全 | `security.msgSecCheck` / `mediaCheckAsync` | 微信官方 API |

### 5.2 云函数清单

| 函数名 | 功能 |
|---|---|
| `user/login` | 登录，获取 openid |
| `dog/create` | 创建狗狗档案 |
| `dog/update` | 更新狗狗资料 |
| `dog/list` | 获取当前用户的狗狗列表 |
| `friend/request` | 发起狗友请求 |
| `friend/confirm` | 确认狗友请求 |
| `friend/list` | 获取狗友列表 |
| `moment/create` | 发布动态（含内容安全检查） |
| `moment/list` | 获取狗友动态流 |
| `moment/like` | 点赞 |
| `moment/comment` | 评论 |
| `walk/create` | 发起约遛 |
| `walk/respond` | 响应约遛 |
| `walk/list` | 约遛列表 |

### 5.3 数据集合

| 集合名 | 说明 | 关键字段 |
|---|---|---|
| `users` | 用户 | openid, nickName, avatarUrl |
| `dogs` | 狗狗档案 | dogId, ownerId, name, avatar, breed, age, gender, tags |
| `friendships` | 狗友关系 | dogId, friendDogId, status(pending/confirmed), createTime |
| `moments` | 动态 | momentId, dogId, content, images[], createTime |
| `comments` | 评论 | commentId, momentId, dogId, content, createTime |
| `likes` | 点赞 | momentId, dogId, createTime |
| `walks` | 约遛 | walkId, dogId, time, location, invitedDogIds[], responses[] |

### 5.4 冷启动影响

云函数闲置后首次调用延迟约 800ms-2s。对"看狗狗圈动态"场景几乎无感。后期可用定时触发器保活。

## 六、合规与上线

### 6.1 主体

北京个体工商户，通过 [e 窗通](https://ect.scjgj.beijing.gov.cn) 注册。

- 经营场所：住宅地址 + 住所承诺申报制
- 经营范围：`信息技术咨询服务`、`软件开发`、`技术服务`
- 费用：0 元
- 耗时：1-3 个工作日

### 6.2 小程序注册

- 主体类型：企业 → 个体工商户
- 认证方式：微信认证（300 元/年）
- 耗时：1-3 个工作日

### 6.3 ICP 备案

- 在微信公众平台内提交，无需域名
- 通管局（北京市通信管理局）审核
- 耗时：7-20 个工作日

### 6.4 类目

**社交-笔记**。所需资质：ICP 备案截图。

### 6.5 代码审核 + 网信办报备

- 代码审核：1-7 天
- 首次社交类目：属地网信办报备，约 7 天
- 预上线 → 正式上线

### 6.6 审核通过必备条件

- 内容安全 API 已接入（文本 + 图片）
- 功能完整，无空壳页面
- 允许先体验再登录（不强制一进入就弹登录）
- 提供测试账号给审核员

### 6.7 上线时间线

### 6.8 风险

| 风险 | 概率 | 缓解 |
|---|---|---|
| 住宅地址注册被驳回 | 低-中 | 修改重提交 |

## 七、边界

**做**：狗狗圈（社交广播）、约遛狗（轻量工具）、微信生态封闭运营。

**不做**：陌生人推荐、IM 聊天、付费交易、电商、直播、广告。
